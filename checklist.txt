Чек-лист проверки веб-приложения (органайзер)
Необходимо проверить следующие функциональные характеристики приложения.
Функциональность:
•	Название задачи
•	Status
•	Deadline
•	Type of Work
•	Estimate
•	Description
•	Comments
•	Move to
•	Общее (настройки/права)
•	Конкурентные ситуации


1.	Название задачи
1.1. Проверить соблюдение уникальности в названии задач в одном проекте клиента
•	одинаковое название 
•	одинаковое название с разным статусом
•	одинаковое название с разным / пустым дедлайном
•	одинаковое название с разным / пустым  значением estimate
•	одинаковое название в разных регистрах
•	пустое название
•	с пробелом в начале названия
•	с цифрой в начале названия
•	с символом в начале названия
•	начало названия в другой раскладке 
•	переместить архивную задачу (завершенную) с таким же названием в проект  
1.2.  проверить возможность наличия задачи с одинаковым названием в  разных проектах клиента
•	одинаковое название, одинаковый статус задачи 
•	одинаковое название, разный статус задачи (возможно лишняя проверка?)
•	одинаковое название, одинаковый дедлайн
•	одинаковое название, одинаковое значение estimate
1.3.  проверить возможность наличия задачи с одинаковым названием в разных проектах  у разных клиентов
•	одинаковое название, одинаковый статус задачи 
•	одинаковое название, одинаковый дедлайн
•	одинаковое название, одинаковое значение estimate
2.	Status
2.1. Добавление и редактирование статусов
•	добавить статус типа «открытый» в настройках
•	добавить статус типа «завершенный» в настройках
•	присвоить новые статусы задачам
•	поменять цвет у статусов в настройках
•	переименовать существующие статусы в настройках
•	проверить изменение названия в статусах задач
2.2. Удаление статусов
•	удалить добавленные статусы в настройках
•	какой статус у задачи с удаленным статусом
2.3. Фильтрация статусов в панели
•	проверить фильтрацию по типу
•	проверить фильтрацию по названию статуса

3.	Deadline
3.1. Выставление даты
•	будущая дата
•	прошедшая дата (позволяет ли)
•	кнопка фиксирования даты OK
•	проверить кнопку today
•	проверить кнопку clear
•	проверить кнопку Cancel
•	проверить знак крестик в окне селектора даты
•	проверить селекторы-стрелки при выборе даты
•	проверить работу клавиш клавиатуры при выборе даты (стрелки, enter, tab) 
3.2.  Изменение цвета шрифта
•	проверить изменение цвета шрифта при приближении даты окончания задачи
•	проверить появление дополнительного значка 
3.3. Влияние соседних полей 
•	проверить функциональность поля при статусе задачи «завершенный»

4.	Type of Work
4.1. Добавление и редактирование типа работы
•	добавить тип задачи в классификацию «оплачиваемая» в настройках
•	добавить  тип задачи в классификацию «неоплачиваемая» в настройках
•	присвоить новые типы задачам
•	переименовать существующий тип задачи в настройках
•	проверить изменение названия типа у задачи
•	изменить тип работы, который присваивается по умолчанию
•	проверить изменение при добавлении новой задачи
4.2. Удаление типа задачи
•	удалить добавленные типы работ в настройках
•	какой тип работы у задачи с удаленным типом
4.3. Архивирование типа работ
•	тип работ добавить в архивный в настройках
•	проверить отображение архивного типа в списке на панели задач
•	как отображается задача с типом работ, ставшим архивным

5.	Estimate
5.1. Ввод времени
•	проверить возможность ввода букв
•	проверить возможность ввода символов
•	проверить возможность ввода пустого значения
•	проверить возможность ввода формата часы:минуты:секунды
•	проверить возможность ввода минимального значения и граничных значений
•	проверить возможность ввода максимального значения и граничных значений
•	проверить очистку введенного времени при нажатии крестика
5.2. Полоса прогресса
•	проверить верность вычисления и отображения полосы прогресса (%)
•	проверить правила окрашивания полосы прогресса согласно требованиям
•	проверить полосу прогресса без введенного значения времени estimate
